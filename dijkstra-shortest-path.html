Shortest Path dengan Algoritma Dijkstra

Visualisasi interaktif untuk memahami cara menemukan jalur terpendek dalam graf berbobot.

Apa itu Shortest Path?

Shortest Path (jalur terpendek) adalah masalah klasik dalam teori graf — menemukan jalur dari satu simpul (node) ke simpul lain dengan total bobot terendah dalam graf berbobot.

Masalah ini sangat umum di dunia nyata: aplikasi peta (Google Maps), routing jaringan, perencanaan logistik, hingga game AI semuanya menggunakan konsep ini.

🗺️

Graf (Graph)

Kumpulan node (simpul) yang dihubungkan oleh edge (sisi/jalur) dengan bobot tertentu.

⚖️

Bobot (Weight)

Nilai numerik pada setiap edge — bisa berupa jarak, waktu, biaya, dll.

🏁

Jalur Terpendek

Urutan node dari sumber ke tujuan dengan total bobot paling minimum.

🔍

Dijkstra

Algoritma greedy yang menjamin jalur terpendek pada graf berbobot non-negatif.

Cara Kerja Algoritma Dijkstra

Algoritma Dijkstra bekerja secara greedy — selalu memilih node dengan jarak terkecil yang belum diproses.

Inisialisasi: Set jarak sumber = 0, semua node lain = ∞. Masukkan semua node ke dalam priority queue.

Pilih node minimum: Ambil node u dengan jarak terkecil dari queue yang belum dikunjungi.

Kunjungi tetangga: Untuk setiap tetangga v dari u, hitung jarak baru = dist[u] + bobot(u,v).

Relaksasi: Jika jarak baru < dist[v], update dist[v] = jarak baru dan catat prev[v] = u.

Tandai dikunjungi: Tandai u sebagai "sudah selesai" agar tidak diproses ulang.

Ulangi langkah 2–5 sampai queue kosong atau semua node terjangkau.

Pseudo-code

DIJKSTRA(Graf G, sumber s):
  // Inisialisasi jarak semua node = tak terhingga
  for setiap node v in G:
      dist[v] = ∞
      prev[v] = NULL
  dist[s] = 0

  Q = semua node G  // priority queue

  while Q tidak kosong:
      u = node di Q dengan dist[u] terkecil
      hapus u dari Q

      for setiap tetangga v dari u:
          alt = dist[u] + bobot(u, v)
          if alt < dist[v]:   // relaksasi
              dist[v] = alt
              prev[v] = u

  return dist[], prev[]

Analisis Kompleksitas

O(V²)
Implementasi array sederhana
O((V+E) log V)
Dengan binary heap / priority queue
O(E + V log V)
Dengan Fibonacci heap (optimal)

V = jumlah vertex (node), E = jumlah edge. Untuk graf jarang (sparse graph), implementasi dengan priority queue jauh lebih efisien.

Kelebihan & Kekurangan

✓ Kelebihan

  • Menjamin jalur terpendek yang optimal
  • Efisien dengan priority queue
  • Mudah diimplementasikan
  • Bisa mencari ke semua node sekaligus

✗ Kekurangan

  • Tidak bisa menangani bobot negatif
  • Lambat untuk graf sangat besar
  • Mengunjungi banyak node yang tidak perlu (tanpa heuristik)
  • Gunakan Bellman-Ford jika ada bobot negatif
